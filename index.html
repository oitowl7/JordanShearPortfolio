Chopman


Chopman/Hangman Vol. 1



  • Choose and interesting theme
  • Use key events (eg: onKeyUp) to listen for letters the player types
  • Display number of wins, the blanked out word with the letters correctly placed if guessed, number of guesses remaining, and letters already guessed


  • Create the images: I had a vision for what I wanted but there wasn't really anything on the internet that fit my needs so I downloaded an image manipulation software called GIMP and used it to create my game images
  • Create the front page: I hard coded the whole front splash page with pretty "manual" css. I had only just been introduced to bootstrap and css frameworks so I was not yet comfortable with it.
  • Began my first Javascript app coding. Really hemmed and hawed over how to make arrays do array things and functions do function things but eventually figured it all out.


  • The biggest challenge for me was making arrays, strings, numbers, and booleans behave the way I wanted to. I quickly learned that it was not the variables that needed to change, but me who needed to change. The elements behave how they do. I needed to adapt to that.
  • I also did not now about the .join(" ").trim command and getting the words to display as I wanted them was not an easy task at the time
  • I also had a pretty bad bug that caused the game to not work right. I never did find out why until well after the game was over but had to do with variables, scoping, and array lengths not being reset. I have left the bug in there for explaination's sake.

Crystals/Cats


Crystal/Cat Game



  • Create a game where each cat has a random, unused value between 1 and 12
  • Randomly generate a target number between 19 and 120 for the user to aim for
  • Direct the user to click each cat to both find the value of each cat and then try to make the user's score add up to the target score
  • Track and display user's wins and losses
  • The primary focus of this assignment was using JQuery to interact with the DOM.


  • Step one was getting all the resources I needed, which was one of the more enjoyable taskes I'v done. It involved looking up cat pictures...for probably far too long.
  • Design the page using HTML/css/bootstrap. It was relatively simple and I just follwed the template I was given.
  • Created the javascript file using JQuery and javascript. It included creating several random numbers and verifying that the number were not previously used in order to avoid duplicate numbers on different cats.


  • The biggest challenge was using JQuery for the firest time. I had not real idea how to properly affect and update the DOM.
  • I also needed to learn how to code my HTML so that it was easier and made more sense to interact with the elements I wanted to affect.
  • I needed to learn how to run an random number generating function and verify that the number was not used before, requiring some functions to run recursively.
  • I also started to get a hint of javascript asyncronicity, but the sequencing of the functions was still quite terrible.

Trivia/Personality Game


Trivia/Personality Game



  • Create a trivia game that shows one question at a time to a player who must answer it before the timer runs out, which is reset upon the question being answered.
  • If the player selects the right answer, show a screen congratulating them on how amazing they are.
  • On the final screen, show the player how many they got right and wrong and give them an option to restart the game without reloading the page.


  • For this project, I went a bit off script from the original instructions, with the blessing of those who would be grading it of course. It mae little sense to me to have a trivia game with only 5-10 pre-scripted questions (we had not yet learned of APIs) so I decided to create a personality test that logged answers and kicked out a character that the user represents.
  • First, I had to decide on a theme. Me being a Star Wars fan, I chose to make my characters about that. I had to choose 22 force sensitive characters from the universe. I then gave each a good/evil value as well as a short bio and stored them all in an array
  • I then created a set of questions and responses. Each response had a value assigned to it from -2 to +2. There was also Jar Jar Binks quotes that were worth 0. If the user selected all Jar Jar quotes or ran out of time on each question, they got Jar Jar as their character. Otherwise, the final score was tallied and their character was shown.


  • The most time consuming part of this project was creating the characters array. I had to find 22 characters (as the user score could be anywhere between -10 and +10 and 2 characters were needed for 0). Finding the characters and creating a very short bio for each character was very time consuming but also fairly enjoyable.
  • Automating the photos that were shown when the final score was tallied was a challenge. I simply ened up naming all the photos the corresponding scores.
  • I wanted to also mix my responses up while maintaining their values. To do this I used random number generators and verifications to randomly display the responses and keep the good/bad values intact.
