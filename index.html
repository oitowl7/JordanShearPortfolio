Case Studies

Chopman


Chopman/Hangman Vol. 1



  • Choose and interesting theme.
  • Use key events (eg: onKeyUp) to listen for letters the player types
  • Display number of wins, the blanked out word with the letters correctly placed if guessed, number of guesses remaining, and letters already guessed


  • Create the images: I had a vision for what I wanted but there wasn't really anything on the internet that fit my needs so I downloaded an image manipulation software called GIMP and used it to create my game images
  • Create the front page: I hard coded the whole front splash page with pretty "manual" css. I had only just been introduced to bootstrap and css frameworks so I was not yet comfortable with it.
  • Began my first Javascript app coding. Really hemmed and hawed over how to make arrays do array things and functions do function things but eventually figured it all out.


  • The biggest challenge for me was making arrays, strings, numbers, and booleans behave the way I wanted to. I quickly learned that it was not the variables that needed to change, but me who needed to change. The elements behave how they do. I needed to adapt to that.
  • I also did not now about the .join(" ").trim command and getting the words to display as I wanted them was not an easy task at the time
  • I also had a pretty bad bug that caused the game to not work right. I never did find out why until well after the game was over but had to do with variables, scoping, and array lengths not being reset. I have left the bug in there for explaination's sake.
